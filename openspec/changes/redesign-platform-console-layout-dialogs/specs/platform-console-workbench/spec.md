## ADDED Requirements

### Requirement: 平台控制台必须提供统一的工作台壳层与主次分区
平台控制台 MUST 在共享壳层中为桌面端提供明确的导航区、页面级标题与操作区、主内容区和辅助信息区，并在窄屏下回落为可用的单列布局。系统 MUST 不得继续让声明为“左侧台账 / 右侧详情”或等价结构的页面在桌面端实际只表现为无主次的纵向堆叠。

#### Scenario: 桌面端进入配置中心、任务中心或系统页
- **WHEN** 用户在桌面宽度下打开 `Config Center`、`Task Center` 或 `System`
- **THEN** 页面必须呈现可辨识的主列表区与详情/编辑区或等价主次结构
- **AND** 用户无需长距离滚动才能在同一视野内理解“当前选中项”和“可执行操作”

#### Scenario: 窄屏进入工作台页面
- **WHEN** 用户在窄屏或移动端进入任一工作台页面
- **THEN** 页面可以回落为单列布局
- **AND** 主操作、当前选中信息和详情内容仍必须保持可达与可理解

### Requirement: 页面首屏必须体现清晰的信息层级和主操作优先级
平台控制台 MUST 在页面首屏明确区分页面定位、当前状态、主操作和辅助操作，不得在侧栏、顶栏和页面头部重复输出同层级信息而稀释首屏重点。系统 MUST 优先让用户先看到最重要的状态、操作与当前上下文，再看到补充说明或次级内容。

#### Scenario: 用户进入总览页
- **WHEN** 用户首次打开总览页
- **THEN** 首屏必须优先显示平台当前关注点、核心指标、最近动态和高优先级入口
- **AND** 不得让重复标题或大量低优先级说明占据首屏主体

#### Scenario: 用户进入任一工作台页面
- **WHEN** 页面同时存在页面标题、顶部状态、主操作按钮和辅助说明
- **THEN** 主操作必须在视觉上优先于辅助说明
- **AND** 辅助信息不得与页面主标题争抢同一层级

### Requirement: 共享控件样式必须在工作台页面中保持一致
平台控制台 MUST 为按钮、输入框、表格、卡片、标签、空状态和操作栏提供统一的视觉和交互状态，包括 hover、active、disabled、focus 和 selected。系统 MUST 不得在同一控制台中让同类控件出现明显不同的圆角、间距、边框和状态反馈口径。

#### Scenario: 用户浏览不同工作台页面
- **WHEN** 用户在首页、配置中心、任务中心、运行中心和系统页之间切换
- **THEN** 相同语义的按钮、输入框、表格选中态和空状态必须保持统一风格
- **AND** 用户无需重新学习同一类操作控件的视觉语义

#### Scenario: 表格行与详情联动
- **WHEN** 用户在列表中选择某一行或悬停某一行
- **THEN** 系统必须以统一方式表现悬停态、选中态和当前详情上下文
- **AND** 选中反馈不得只依赖细微颜色变化而难以识别
