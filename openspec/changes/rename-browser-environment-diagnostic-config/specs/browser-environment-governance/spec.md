## ADDED Requirements

### Requirement: 仓库对外语义必须统一为浏览器环境配置
系统 MUST 在 UI、API、PRD、设计文档、运行报告和操作员文案中统一使用“浏览器环境配置”或等价合规命名来描述该能力，不得继续以“设备指纹伪装”“反检测”“stealth”或“绕过保护”作为对外目标语义。历史术语仅可出现在迁移说明或弃用说明中，并且 MUST 显式标注为历史上下文。

#### Scenario: 查看配置中心和方案编辑器
- **WHEN** 操作员在控制台中查看配置中心、方案编辑器或运行详情
- **THEN** 系统 MUST 使用“浏览器环境配置”及等价合规命名，而不是继续暴露 `fingerprint`、`stealth` 或 `spoof` 语义

#### Scenario: 查看 API 和报告文案
- **WHEN** 操作员查看 API 返回结构、报告摘要或下载归档说明
- **THEN** 系统 MUST 使用统一的浏览器环境配置命名，并避免把能力描述成降低检测率或绕过保护

### Requirement: 历史指纹配置资产必须可迁移或受控淘汰
系统 MUST 为历史“指纹配置”模板、导出 JSON、文档和界面入口提供迁移或受控淘汰路径。对只包含白名单字段的历史资产，系统 MUST 支持映射到新的浏览器环境配置模型并保留迁移来源；对包含禁止字段、禁止语义或不可映射内容的历史资产，系统 MUST 阻断迁移并记录明确原因。

#### Scenario: 迁移只包含白名单字段的历史资产
- **WHEN** 操作员导入一个只包含白名单字段的历史指纹配置资产
- **THEN** 系统 MUST 将其映射为浏览器环境配置，并保留原始来源和迁移说明

#### Scenario: 迁移包含禁止字段的历史资产
- **WHEN** 操作员导入一个包含 `stealthMode`、噪声注入或其他禁止内容的历史指纹配置资产
- **THEN** 系统 MUST 拒绝迁移，并返回可读的阻断原因和后续处理提示

### Requirement: 检测站评分语义必须从验收主链路移除
系统 MUST 将 `bot.sannysoft.com` 等检测站从正式验收链路和主评分语义中移除；如果仓库保留相关文档或引用，它们 MUST 被标识为补充诊断材料，而不是能力通过标准。

#### Scenario: 查看正式验收说明
- **WHEN** 操作员查看浏览器环境配置的正式验收说明或交付文档
- **THEN** 系统 MUST 以授权站点测试结果、配置回显和审计记录说明验收标准，而不是引用检测站评分

#### Scenario: 查看保留的历史检测站参考
- **WHEN** 仓库中仍保留检测站相关的历史参考链接或迁移说明
- **THEN** 这些内容 MUST 明确标注为补充诊断或历史背景，而不是当前必经流程
