## ADDED Requirements

### Requirement: 运行前必须按白名单应用浏览器环境配置
系统 MUST 在任务启动前把方案绑定的浏览器环境配置解析为运行时设置，并且只允许 `UA`、`UA-CH`、`locale`、`timezone`、`viewport`、`screen`、`geolocation`、浏览器版本及其等价受控字段进入运行时应用链路。系统 MUST 不得在该链路中启用 `stealthMode`、Canvas/WebGL/Audio 噪声、插件伪造、`navigator` 深度篡改或随机轮换逻辑。

#### Scenario: 使用已绑定的浏览器环境配置启动运行
- **WHEN** 操作员启动一个已绑定浏览器环境配置的测试方案
- **THEN** 系统 MUST 在运行前应用该配置中的白名单字段，并将实际生效的配置摘要记录到运行上下文

#### Scenario: 运行链路命中禁止能力
- **WHEN** 运行准备阶段发现浏览器环境配置请求启用了禁止字段或禁止逻辑
- **THEN** 系统 MUST 在执行前阻断本次运行，并明确说明被拒绝的原因

### Requirement: 运行前必须执行配置一致性与审批校验
系统 MUST 在执行前校验浏览器环境配置是否具备必需字段、是否处于允许执行的审批状态、以及关键字段之间是否自相矛盾；校验失败时 MUST Fail Fast，而不是带着不一致配置继续执行。

#### Scenario: 浏览器环境配置缺少必需字段
- **WHEN** 方案绑定的浏览器环境配置缺少必需字段或字段格式非法
- **THEN** 系统 MUST 在任务启动前拒绝执行，并返回明确的校验失败信息

#### Scenario: 浏览器环境配置未通过审批
- **WHEN** 方案绑定的浏览器环境配置没有满足组织要求的审批状态
- **THEN** 系统 MUST 在执行前阻断运行，并提示需要更换或审批该配置

### Requirement: 运行详情与报告必须回显环境配置和审计信息
系统 MUST 在运行详情、报告摘要和等价产物元数据中回显浏览器环境配置的名称、版本、来源、审批状态、方案绑定关系和实际生效的白名单字段摘要，使操作员能够复核本次运行使用的测试环境。

#### Scenario: 查看运行详情中的环境配置回显
- **WHEN** 操作员打开某次运行的详情页面
- **THEN** 系统 MUST 展示本次运行绑定的浏览器环境配置及其来源、审批状态和有效字段摘要

#### Scenario: 查看报告中的环境配置审计摘要
- **WHEN** 操作员查看一份与运行相关的报告或归档摘要
- **THEN** 系统 MUST 提供该运行对应的浏览器环境配置回显和审计信息，而不是只展示原始日志

### Requirement: 验收判断必须以授权站点结果和审计记录为主
系统 MUST 以授权站点测试结果、人工接续状态、浏览器环境配置回显和审计记录作为该能力的主要验收依据。系统 MUST 不得把 `bot.sannysoft.com` 等检测站评分作为运行通过/失败或能力有效/无效的主链路判定条件。

#### Scenario: 生成一次可验收的运行摘要
- **WHEN** 一次授权站点测试运行结束
- **THEN** 系统 MUST 能基于站点结果、人工接续状态、配置回显和审计记录生成可验收的摘要结论

#### Scenario: 检测站结果仅作为补充诊断
- **WHEN** 系统存在历史检测站链接、文档或手工诊断记录
- **THEN** 这些信息 MUST 被视为补充诊断材料，而不是本次能力验收的必要评分项
