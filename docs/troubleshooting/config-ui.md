# 配置界面故障排除指南

## 问题：点击"开始测试"按钮无反应

### 可能的原因和解决方案

#### 1. 已有测试任务正在运行

**症状：**
- 点击"开始测试"按钮没有任何反应
- 状态栏没有显示错误信息

**原因：**
配置服务器同时只能运行一个测试任务。如果之前的测试任务还在运行或没有正确清理，新的测试无法启动。

**解决方案：**

方法 1：通过界面停止
1. 刷新浏览器页面
2. 查看"运行状态"面板
3. 如果显示"运行中"，点击"停止测试"按钮
4. 等待状态变为"已停止"
5. 再次点击"开始测试"

方法 2：通过 API 停止
```bash
curl -X POST http://127.0.0.1:3200/api/run/stop
```

方法 3：重启配置服务器
```bash
# 停止配置服务器（Ctrl+C）
# 重新启动
npm run config:ui
```

#### 2. 配置服务器未运行

**症状：**
- 点击按钮后浏览器控制台显示网络错误
- 页面无法加载或显示"连接失败"

**解决方案：**
```bash
# 启动配置服务器
npm run config:ui
```

#### 3. 测试站点地址未配置

**症状：**
- 点击按钮后显示错误："请输入完整且有效的地址"

**解决方案：**
1. 在"配置测试站点地址"区域输入完整的 HTTPS 地址
2. 点击"保存配置"
3. 确认"当前配置预览"显示正确的地址
4. 再次点击"开始测试"

#### 4. JavaScript 错误

**症状：**
- 按钮点击无反应
- 浏览器控制台显示 JavaScript 错误

**解决方案：**
1. 打开浏览器开发者工具（F12）
2. 切换到"Console"标签
3. 查看是否有红色错误信息
4. 刷新页面清除缓存（Ctrl+Shift+R）
5. 如果错误持续，重启配置服务器

## 调试步骤

### 1. 检查配置服务器状态

```bash
# 检查服务器是否运行
curl http://127.0.0.1:3200/health

# 预期输出：
# {"ok":true}
```

### 2. 检查当前配置

```bash
# 获取当前配置
curl http://127.0.0.1:3200/api/config

# 预期输出包含 startUrl
```

### 3. 检查运行状态

```bash
# 获取当前运行状态
curl http://127.0.0.1:3200/api/run

# 检查 status 字段：
# - "idle" = 空闲，可以启动新测试
# - "running" = 正在运行，需要先停止
# - "stopping" = 正在停止
# - "passed" / "failed" / "stopped" = 已完成
```

### 4. 手动启动测试

```bash
# 通过 API 启动测试
curl -X POST http://127.0.0.1:3200/api/run \
  -H "Content-Type: application/json" \
  -d '{"mode":"headless"}'

# 如果成功，会返回：
# {"ok":true,"message":"测试任务已启动。","run":{...}}

# 如果失败，会返回错误信息：
# {"ok":false,"message":"错误原因","run":{...}}
```

### 5. 查看浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 切换到"Console"标签
3. 点击"开始测试"按钮
4. 查看是否有错误信息

常见错误：
- `Failed to fetch` - 配置服务器未运行
- `Unexpected token` - JSON 解析错误
- `undefined is not a function` - JavaScript 代码错误

### 6. 查看网络请求

1. 打开浏览器开发者工具（F12）
2. 切换到"Network"标签
3. 点击"开始测试"按钮
4. 查看 `/api/run` 请求的状态

检查项：
- 请求是否发送成功（状态码 200 或 400）
- 响应内容是否包含错误信息
- 请求是否超时

## 常见错误信息

### "当前已有测试任务正在运行"

**原因：** 之前的测试任务还在运行

**解决：** 点击"停止测试"或等待测试完成

### "请输入完整且有效的地址"

**原因：** 测试站点地址未配置或格式不正确

**解决：** 配置正确的 HTTPS 地址

### "启动测试失败"

**原因：** 可能是环境配置问题或依赖缺失

**解决：**
1. 检查 `.env` 文件是否配置正确
2. 运行 `npm install` 确保依赖已安装
3. 检查 Playwright 浏览器是否已安装：`npx playwright install`

## 完整的测试流程

1. **启动配置服务器**
   ```bash
   npm run config:ui
   ```

2. **打开浏览器**
   访问：http://127.0.0.1:3200

3. **配置测试站点**
   - 输入测试站点地址（必须是 HTTPS）
   - 点击"保存配置"
   - 确认"当前配置预览"显示正确

4. **配置临时邮箱（可选）**
   - 输入临时邮箱 API 地址和密钥
   - 点击"测试连接"验证配置
   - 点击"保存临时邮箱配置"

5. **启动测试**
   - 选择运行模式（无头/可视）
   - 点击"开始测试"
   - 查看实时日志

6. **查看结果**
   - 等待测试完成
   - 点击"打开测试报告"查看详细结果

## 重置配置服务器

如果所有方法都无效，可以尝试完全重置：

```bash
# 1. 停止配置服务器（Ctrl+C）

# 2. 清理进程（Windows）
taskkill /F /IM node.exe /T

# 3. 删除配置文件（可选）
rm -rf config/

# 4. 重新启动
npm run config:ui
```

## 获取帮助

如果问题仍然存在，请提供以下信息：

1. 浏览器控制台的完整错误信息
2. 配置服务器的输出日志
3. `/api/run` 请求的响应内容
4. 操作系统和浏览器版本

可以通过以下命令收集诊断信息：

```bash
# 收集系统信息
echo "=== System Info ===" > debug.log
node --version >> debug.log
npm --version >> debug.log

# 收集配置信息
echo "=== Config ===" >> debug.log
curl -s http://127.0.0.1:3200/api/config >> debug.log

# 收集运行状态
echo "=== Run Status ===" >> debug.log
curl -s http://127.0.0.1:3200/api/run >> debug.log

# 查看日志
cat debug.log
```
