# 项目架构说明

## 概述

这是一个基于 Playwright 的自动化测试框架，专注于验证目标站点的保护机制（CAPTCHA、短信验证、设备挑战等）是否正常工作。项目遵循"安全边界明确"的原则，只验证保护机制，不提供任何绕过手段。

## 核心设计原则

1. **安全边界明确** - 只检测保护机制，不绕过
2. **配置驱动** - 通过配置文件控制行为
3. **模块化设计** - 功能独立，易于扩展
4. **可观测性** - 完整的日志、截图、trace 记录

## 模块详解

### 1. 配置层

**职责：** 管理所有配置信息，提供统一的配置接口

**组件：**
- `.env` - 环境变量（IMAP 配置、浏览器设置等）
- `config/target-site.json` - 目标站点 URL
- `config/temp-mail.json` - 临时邮箱域名和 API 配置
- `src/target.profile.ts` - 目标站点的字段选择器和期望结果

**设计要点：**
- 敏感信息通过 `.env` 管理，不提交到 git
- JSON 配置文件便于程序读写
- TypeScript 配置文件提供类型安全

### 2. Web 配置界面

**职责：** 提供可视化的配置和测试管理界面

**组件：**
- `scripts/config-server.mjs` - Express HTTP 服务器
- `src/config-ui/` - 前端 HTML/CSS/JS

**功能：**
- 配置目标站点 URL
- 选择运行模式（有头/无头）
- 启动/停止测试
- 实时查看日志
- 打开测试报告

### 3. 核心测试引擎

**职责：** 执行自动化测试流程

**主要文件：**
- `tests/protection-validation.spec.ts` - 主测试用例
- `playwright.config.ts` - Playwright 配置

**测试流程：**
1. 加载配置
2. 启动浏览器（应用隐身技术）
3. 访问目标站点
4. 填充表单字段
5. 检测保护机制
6. 处理邮箱验证码
7. 生成测试报告

### 4. 保护检测模块

**职责：** 检测各种保护机制是否触发

**文件：** `src/protection.ts`

**检测类型：**
- CAPTCHA（reCAPTCHA、hCaptcha、Cloudflare Turnstile）
- 短信验证挑战
- 设备安全挑战
- 阻断页面

**检测方法：**
- DOM 元素检测
- iframe 检测
- 特征文本匹配
- URL 模式匹配

### 5. 邮箱验证模块

**职责：** 获取和提取邮箱验证码

**目录：** `src/email/`

**支持方式：**
- **IMAP 邮箱** - 连接真实邮箱，轮询新邮件
- **临时邮箱 API** - 使用 1secmail.com 等服务

**工作流程：**
1. 生成/使用邮箱地址
2. 轮询新邮件
3. 解析邮件内容
4. 正则提取验证码
5. 返回验证码供测试使用

### 6. 隐身技术模块

**职责：** 降低自动化检测特征

**目录：** `src/stealth/`

**技术手段：**
- 隐藏 WebDriver 特征
- 随机化浏览器指纹
- 处理权限请求
- 清理自动化痕迹

**注意：** 这些技术用于降低误报，不是为了绕过保护机制

### 7. 工具层

**职责：** 提供通用工具函数

**组件：**
- `src/utils/` - 工具函数集合
- `src/env.ts` - 环境变量加载和验证
- `src/types.ts` - TypeScript 类型定义

## 数据流

```
用户配置 → Web UI / .env → 配置加载器
                              ↓
                         测试引擎启动
                              ↓
                    浏览器启动（应用隐身）
                              ↓
                         访问目标站点
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
              填充表单字段          检测保护机制
                    ↓                   ↓
              需要邮箱验证？        发现保护？
                    ↓                   ↓
              获取验证码            记录并处理
                    ↓                   ↓
              继续流程              生成报告
                    ↓                   ↓
                    └─────────┬─────────┘
                              ↓
                      生成测试报告
                              ↓
                  截图 / trace / 视频
```

## 测试策略

### 单元测试
- 工具函数测试
- 配置加载测试
- 验证码提取测试

### 集成测试
- 邮箱服务集成测试（`tests/debug-tempmail.spec.ts`）
- 配置服务器测试

### 端到端测试
- 完整流程测试（`tests/protection-validation.spec.ts`）

## 性能考虑

1. **浏览器启动优化** - 使用本地浏览器，避免下载
2. **并发控制** - 单个测试实例，避免资源竞争
3. **超时设置** - 合理的超时时间，避免长时间等待
4. **资源清理** - 测试结束后清理浏览器进程

## 安全考虑

1. **敏感信息保护** - `.env` 不提交到 git
2. **权限最小化** - 只请求必要的浏览器权限
3. **输入验证** - 验证所有配置输入
4. **错误处理** - 避免泄露敏感信息到日志
