# 前端配置界面 - 临时邮箱功能使用指南

## 功能概述

前端配置界面现已支持临时邮箱的可视化配置，无需手动编辑 `.env` 文件。

## 新增功能

### 1. 临时邮箱配置面板

在配置界面中新增了"临时邮箱配置"面板，包含：
- ✅ API 地址配置
- ✅ API Key 配置
- ✅ 一键保存
- ✅ 连接测试

### 2. 自动更新环境变量

保存配置后，系统会自动：
- 创建 `config/temp-mail.json` 配置文件
- 更新 `.env` 文件中的相关配置
- 设置 `USE_TEMP_MAIL=true`

## 使用步骤

### 第一步：启动配置界面

```bash
npm run config:ui
```

浏览器访问：http://127.0.0.1:3200

### 第二步：配置临时邮箱

在"临时邮箱配置"面板中：

1. **填写 API 地址**
   ```
   http://114.215.173.42:888
   ```

2. **填写 API Key**
   ```
   tm_admin_36f53ee440748349007538fde32d1aeeb3ac028c804d7b90
   ```

3. **点击"测试连接"**
   - 系统会创建一个临时邮箱
   - 验证 API 是否可用
   - 显示测试邮箱地址
   - 自动删除测试邮箱

4. **点击"保存临时邮箱配置"**
   - 保存到 `config/temp-mail.json`
   - 自动更新 `.env` 文件
   - 启用临时邮箱功能

### 第三步：运行测试

1. 在"配置测试站点地址"面板填写目标网站
2. 选择运行模式（无头/可视）
3. 点击"开始测试"

测试将自动：
- 创建临时邮箱
- 填充邮箱地址
- 接收验证码
- 完成注册/登录
- 删除临时邮箱

## 错误处理

### 连接测试失败

**错误信息：**
```
连接失败: API 返回错误: 401 Unauthorized
```

**解决方案：**
- 检查 API Key 是否正确
- 确认 API 地址可访问
- 验证网络连接

### 保存配置失败

**错误信息：**
```
保存临时邮箱配置时发生未知错误。
```

**解决方案：**
- 检查 `config` 目录是否存在
- 确认有写入权限
- 查看服务器日志

### 测试运行时邮箱创建失败

**错误信息：**
```
[TempMail] Failed to create mailbox: 401 Unauthorized
```

**解决方案：**
- 重新保存临时邮箱配置
- 确认 `.env` 文件已更新
- 重启配置服务器

## 常见问题

### Q1: 配置保存后，测试时还是用的旧邮箱？

**A:** 需要重启配置服务器：
```bash
# Ctrl+C 停止
npm run config:ui
```

### Q2: 可以同时使用 IMAP 和临时邮箱吗？

**A:** 不可以。系统会根据 `USE_TEMP_MAIL` 环境变量选择：
- `USE_TEMP_MAIL=true` - 使用临时邮箱
- `USE_TEMP_MAIL=false` - 使用 IMAP

### Q3: 临时邮箱配置存储在哪里？

**A:** 两个位置：
1. `config/temp-mail.json` - 配置文件
2. `.env` - 环境变量（自动同步）

### Q4: 测试连接成功，但运行测试时失败？

**A:** 可能原因：
1. 配置服务器未重启
2. `.env` 文件未更新
3. API 服务暂时不可用

**解决方案：**
```bash
# 1. 检查 .env 文件
cat .env | grep TEMP_MAIL

# 2. 重启配置服务器
npm run config:ui

# 3. 重新运行测试
npm test
```
