# 实时日志查看器使用指南

## 🎯 功能介绍

实时日志查看器是一个基于 WebSocket 的前端界面，可以实时显示 Playwright 测试的日志输出，帮助您快速定位问题。

## ✨ 特性

- ✅ **实时日志流** - WebSocket 实时传输日志
- ✅ **彩色分类** - 不同级别的日志用不同颜色显示
- ✅ **智能过滤** - 按级别、标签、关键词过滤
- ✅ **搜索高亮** - 搜索关键词自动高亮
- ✅ **自动滚动** - 可选的自动滚动到最新日志
- ✅ **统计信息** - 实时显示各类日志数量
- ✅ **导出功能** - 一键导出日志到文件
- ✅ **优雅界面** - VS Code 风格的暗色主题

## 🚀 快速开始

### 1. 安装依赖

```bash
npm install
```

这会安装以下新增依赖：
- `ws` - WebSocket 服务器
- `@types/ws` - TypeScript 类型定义
- `ts-node` - 运行 TypeScript 脚本

### 2. 启动日志服务器

```bash
npm run log:server
```

您会看到：
```
========================================
实时日志服务器已启动
========================================

📊 日志查看器: http://localhost:3101
🔌 WebSocket 端点: ws://localhost:3100

在浏览器中打开日志查看器，然后点击"开始测试"按钮
```

### 3. 打开日志查看器

在浏览器中访问：
```
http://localhost:3101
```

或者使用快捷命令（会自动打开浏览器）：
```bash
npm run log:viewer
```

### 4. 开始测试

在日志查看器界面中：
1. 等待连接成功（状态栏显示"已连接到服务器"）
2. 点击 **▶ 开始测试** 按钮
3. 实时查看日志输出

## 📖 界面说明

### 顶部控制栏

```
🔍 实时日志监控
[▶ 开始测试] [⏹ 停止] [🗑 清空日志] [💾 导出日志]
```

- **开始测试** - 启动 Playwright 测试并开始接收日志
- **停止** - 停止当前运行的测试
- **清空日志** - 清空当前显示的所有日志
- **导出日志** - 将日志导出为文本文件

### 状态栏

显示当前状态和运行时间：
```
运行中... | 00:45
```

状态颜色：
- 🔵 蓝色 - 正在运行
- 🟢 绿色 - 测试成功
- 🔴 红色 - 测试失败/错误
- 🟡 黄色 - 已停止/警告

### 左侧边栏

#### 搜索框
输入关键词实时搜索日志，匹配的内容会高亮显示。

#### 日志级别过滤
- ℹ️ **信息** - 一般信息日志（蓝色）
- ✅ **成功** - 成功操作日志（绿色）
- ⚠️ **警告** - 警告信息（黄色）
- ❌ **错误** - 错误信息（红色）
- 🐛 **调试** - 调试信息（灰色）

#### 日志标签过滤
根据日志标签（如 `[TempMail]`, `[Flow]`）进行过滤。

#### 选项
- **自动滚动** - 新日志到达时自动滚动到底部
- **显示时间戳** - 显示/隐藏每条日志的时间戳

### 主日志区域

显示实时日志，每条日志包含：
```
[时间戳] [标签] 日志内容
```

示例：
```
10:30:45 [TempMail] Created temporary email: test@example.com
10:30:46 [Flow] Email verification code input detected
10:30:47 [TempMail] ✓ Extracted verification code: 123456
```

### 底部统计栏

实时显示日志统计：
```
总计: 150 | 信息: 100 | 成功: 30 | 警告: 15 | 错误: 5
```

## 🔍 使用场景

### 场景 1: 调试验证码获取问题

1. 启动日志服务器
2. 打开日志查看器
3. 点击"开始测试"
4. 观察日志输出：

```
[TempMail] Created temporary email: xxx@xxx.com
[Flow] Email verification code input detected
[TempMail] Waiting for verification email...
[TempMail] Polling attempt 1...
[TempMail] No email found yet
[TempMail] Polling attempt 2...
[TempMail] ✓ Got latest email: from=no-reply@openai.com
[TempMail] ✓ Extracted verification code: 123456
```

**如果卡住：**
- 查看最后一条日志
- 使用搜索功能搜索 "error" 或 "failed"
- 检查是否一直在 "Polling attempt" 循环

### 场景 2: 监控完整测试流程

1. 启用所有日志级别
2. 关闭过滤器
3. 开始测试
4. 观察完整流程：
   - 邮箱创建
   - 邮箱地址填写
   - 验证码获取
   - 验证码填写
   - 密码填写
   - 表单提交

### 场景 3: 导出日志用于分析

1. 运行测试直到完成或出错
2. 点击"导出日志"按钮
3. 保存为文本文件
4. 发送给技术支持或自行分析

## 🎨 日志颜色说明

| 级别 | 颜色 | 边框 | 用途 |
|------|------|------|------|
| 信息 | 蓝色 | 蓝色 | 一般流程信息 |
| 成功 | 绿色 | 绿色 | 成功的操作 |
| 警告 | 黄色 | 黄色 | 需要注意的信息 |
| 错误 | 红色 | 红色 | 错误和失败 |
| 调试 | 灰色 | 灰色 | 调试信息 |

## 💡 使用技巧

### 1. 快速定位错误

在搜索框输入：
- `error` - 查找所有错误
- `failed` - 查找失败信息
- `✗` - 查找失败标记

### 2. 监控特定模块

使用标签过滤器：
- 只勾选 `[TempMail]` - 只看邮件相关日志
- 只勾选 `[Flow]` - 只看流程相关日志

### 3. 性能分析

1. 搜索 "elapsed" 查看耗时
2. 搜索 "attempt" 查看重试次数
3. 导出日志后用工具分析时间戳

### 4. 对比测试

1. 运行第一次测试
2. 导出日志为 `test1.txt`
3. 清空日志
4. 运行第二次测试
5. 导出日志为 `test2.txt`
6. 使用 diff 工具对比

## 🔧 高级配置

### 修改端口

编辑 `scripts/log-server.ts`：

```typescript
const PORT = 3100;           // WebSocket 端口
const LOG_VIEWER_PORT = 3101; // HTTP 端口
```

### 自定义日志级别

编辑 `scripts/log-server.ts` 中的 `parseLogLevel` 函数：

```typescript
function parseLogLevel(message: string): string {
  if (message.includes('YOUR_KEYWORD')) {
    return 'success';
  }
  // ...
}
```

### 添加自定义过滤器

编辑 `public/log-viewer.html`，在侧边栏添加新的过滤选项。

## 🐛 故障排除

### 问题 1: 无法连接到服务器

**症状：** 状态栏显示"连接已断开"

**解决方案：**
1. 确认日志服务器正在运行
2. 检查端口 3100 和 3101 是否被占用
3. 查看服务器控制台是否有错误

### 问题 2: 日志不更新

**症状：** 点击"开始测试"后没有日志输出

**解决方案：**
1. 检查 WebSocket 连接状态
2. 打开浏览器开发者工具查看控制台
3. 确认测试是否真的在运行

### 问题 3: 页面卡顿

**症状：** 日志很多时页面变慢

**解决方案：**
1. 使用过滤器减少显示的日志
2. 定期清空日志
3. 关闭自动滚动

## 📊 架构说明

```
┌─────────────────┐
│  浏览器         │
│  public/log-viewer.html│
└────────┬────────┘
         │ WebSocket
         │ ws://localhost:3100
         │
┌────────▼────────┐
│  日志服务器      │
│  log-server.ts  │
└────────┬────────┘
         │ spawn
         │
┌────────▼────────┐
│  Playwright     │
│  测试进程        │
└─────────────────┘
```

工作流程：
1. 浏览器连接到 WebSocket 服务器
2. 用户点击"开始测试"
3. 服务器启动 Playwright 测试进程
4. 捕获测试的 stdout/stderr
5. 通过 WebSocket 实时发送到浏览器
6. 浏览器解析并显示日志
